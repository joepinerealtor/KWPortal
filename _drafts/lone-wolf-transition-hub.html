Step Guides

Detailed lessons with the exact steps agents will follow

Lesson 1

Connect the Lone Wolf account in Command

Use this lesson when an agent is setting up access for the first time or reconnecting an existing Lone Wolf account.

Setup Command Settings
  1. Log into Command.Use the standard Keller Williams login at `agent.kw.com` so the setup happens inside the correct Command profile.
  2. Open Settings from the profile menu.At the top-right of Command, click the profile picture and choose Settings.
  3. Find Lone Wolf under Connected Applications.In the Digital Signature & Transactions area, select `Connect Account` for Lone Wolf.
  4. Sign up or sign in.If the agent already has a Lone Wolf product account, they should sign in with it. If not, they can create the free Command Edition account.
  5. Verify the email and finish setup.After verification, the account is ready for forms access and transaction creation from Opportunities.
Open KW Guide

Lesson 2

Prepare the Opportunity and create the linked transaction

This is the Command-side setup that makes the rest of the workflow cleaner, especially when the office checklist structure matters.

Command Checklist Type
  1. Create or find the contact.Make sure the buyer, seller, tenant, or landlord exists in Command before building the Opportunity.
  2. Build the Opportunity with as much detail as possible.Dates, people, property details, and dollar amounts all help later when mapping data into Transact.
  3. Open the Documents tab.Once inside the Opportunity, go to `Documents` to start the compliance-connected workflow.
  4. Select the right checklist type.The checklist type is tied to Market Center setup, so it determines the document expectations and can also control linked forms.
  5. Click `Sync with Transact`.If multiple providers are connected, choose Lone Wolf Transact from the provider menu to create the linked transaction.
Create Transaction Guide Workflow Overview

Lesson 3

Open the transaction, prep forms, and send for signature

Once the sync is done, this is the Transact-and-Authentisign portion of the flow.

Transact Authentisign
  1. Use `Go to Transact` from the Opportunity.Return to the Opportunity Documents tab and open the linked transaction directly from there.
  2. Review the transaction details and parties.Update listing information, add contacts or vendors, and fill in any gaps that were not mapped from Command.
  3. Add the needed forms and documents.Use the form library, templates, and any office-provided structure to build the working packet.
  4. Complete the unsigned portions of the forms.Finish as much of the paperwork as possible before asking for signatures, including reusable clauses when needed.
  5. Create the signing in Authentisign.Assign recipients, make final checks, and send the packet out for email signature.
Open Transaction Guide Authentisign Overview

Lesson 4

Attach the signed files back to the Opportunity

Command still owns the compliance handoff, so the signed PDFs have to come back into the right event and checklist items.

Documents Tab Compliance Events
  1. Return to Documents in the Opportunity.Make sure the correct checklist type is still selected before attaching anything.
  2. Pick the compliance event.Choose `Listed`, `Under Contract`, `Closed`, or the event that matches where the transaction is in the process.
  3. Review required and conditional documents.Use the checklist so the agent knows exactly what needs to be present before submission.
  4. Use `Attach Files From`.Pull the signed file from the connected Lone Wolf transaction, another provider, a custom folder, or upload the PDF manually.
  5. Match each file to the right checklist item.Keep attaching until the event is fully documented and ready to move forward.
Attach Documents Guide

Lesson 5

Submit to the Market Center and handle returns

This closes the workflow loop and gives agents the basic playbook for corrections when a document comes back.

Submit to MC Returned Docs
  1. Choose the completed compliance event.Inside Documents, select the event that is fully assembled and ready for review.
  2. Click `Submit to MC`.Confirm the submission once all required files are present and attached correctly.
  3. Track the status after submission.Documents can be approved or returned depending on what the Market Center review finds.
  4. Use `Update` when a file needs correction.If something is wrong, replace the document on the checklist and resubmit after the correction is made.
  5. Watch for rejection comments.Use the returned status and comment details to understand exactly what needs to be fixed before sending it back.

Launch Notes

How this draft can grow from training page to live resource hub

Before Go-Live

Build the page

Finish gathering the workflow articles, office expectations, and first-wave support material before the office publicly points agents here.

  • Confirm which links should stay at the top of the page.
  • Add office-specific FAQs, dates, and contact paths.
  • Decide which internal training recording should be the featured resource.
Go-Live Week

Use it daily

This draft can become the office help center during the first wave of adoption once the rollout date is announced.

  • Pin the most common troubleshooting steps and training sessions.
  • Update the FAQ based on what agents are asking in real time.
  • Keep the workflow articles visible so agents can self-serve.
After Launch

Keep it current

Once the transition settles, this can remain the permanent Transact resource page rather than a one-time announcement page.

  • Retire outdated launch notes and keep evergreen guidance.
  • Refresh videos, SOPs, and templates as the workflow evolves.
  • Link it into the portal only when the office is ready for it to be a standing page.
